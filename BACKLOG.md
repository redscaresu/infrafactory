| M61 | Maintenance | fakeaws RDS DB instance apply-loop hang isolation. Closed by fakeaws@pending + infrafactory@pending. Root cause was multi-part — TF_LOG trace showed (a) the 4th DescribeDBInstances call after Create uses `Filters.Filter.N.Name=dbi-resource-id&Filters.Filter.N.Values.Value.M=db-<hex>` (the provider rounds-trips the create-response DbiResourceId as a follow-up lookup key); fakeaws had derived DbiResourceId from the user identifier (`"db-" + inst.ID`), so the lookup hit our identifier-keyed path with a mangled id and 404'd. (b) DeleteDBInstance returned a bare envelope (nil payload); SDK errored with `DeleteDBInstanceResult node not found, failed to decode response body`.
